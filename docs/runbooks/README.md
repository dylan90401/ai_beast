## Runbook Structure

Each runbook follows this structure:

1. **Overview** - What this runbook covers
2. **Prerequisites** - What you need before starting
3. **Procedure** - Step-by-step instructions
4. **Verification** - How to verify success
5. **Rollback** - How to undo changes
6. **Related** - Links to related runbooks

## Emergency Contacts

For urgent issues, contact:

- **On-Call**: Check your team's on-call schedule
- **Escalation**: Follow your organization's escalation policy
